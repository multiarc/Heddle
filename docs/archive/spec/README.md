# Heddle implementation specifications

Detailed technical specifications for executing the
[evolution roadmap](../roadmap/README.md), one spec per phase. The roadmap documents say
*what* each phase does and *why* (ratified designs, success criteria, grounding); the
specs here say **exactly how** — file-level implementation plans, final API contracts,
closed decision records, diagnostics, and concrete test assets — with **no open
questions** left for the implementer.

Phases are specified and implemented **sequentially (1 → 9), each as a separate
effort**, and the sequence culminates in **one combined v2.0 release** — all nine phases
together, with the breaking deltas consolidated in the
[2.0 migration spec](common/migration-2.0.md) and amendments discovered during
implementation absorbed through the
[ledger](common/cross-cutting-decisions.md#cross-spec-amendments-ledger). Every spec
assumes the previous phases are merged and binds to their APIs. See the
[spec conventions](common/spec-conventions.md) for the full rule set governing these
documents. Maintainer-level unresolved items live only in the
[open-questions register](OPEN-QUESTIONS.md), each with a provisional default the specs
implement.
