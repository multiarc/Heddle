# Historical records

The append-only record of completed spec work. The [common specs](common/spec-conventions.md)
are instruction-only — anything that documents *finished* work (per-window as-shipped
records, the accumulated amendments ledger, registry corrections) lives here instead.
Entries are never edited or deleted; later facts are recorded as new entries.

> **Retired-document citations.** Entries created during the v2.0 effort cite its phase
> specs (`phase-1` … `phase-9`), its roadmap, and its open-questions register. Those
> documents are removed from the tree (available in git history); citations are kept as
> plain text for provenance. Where a retired spec's content landed in a published docs
> page, the living page is linked instead.

## The 2.0 breaking window — as-shipped record

The first window under the [breaking-windows policy](common/breaking-windows.md),
reconciled against the released 2.0.0 source (2026-07).

**Specified contents (five items, as ratified by the v2.0 effort) with as-shipped status:**

| # | Change | 1.x state | Specified 2.0 state | **As shipped in 2.0.0** |
| --- | --- | --- | --- | --- |
| 1 | `OutputProfile` default flip | `Text` | `Html` — the unnamed `@(...)` encodes by default | ✅ Shipped ([TemplateOptions.cs](../../src/Heddle/Data/TemplateOptions.cs) ctor) |
| 2 | Encoder swap + pluggability | `WebUtility.HtmlEncode` at the pinned call sites; string-based span bridge | `HtmlEncoder.Create(UnicodeRanges.All)` default; new `TemplateOptions.Encoder`; `HtmlEncodedRenderer` slots `TextEncoder.Encode`/`EncodeUtf8` | ❌ **Slipped** — 2.0.0 still encodes via `WebUtility` and has no `Encoder` option ([HtmlEncodedRenderer.cs](../../src/Heddle/Data/HtmlEncodedRenderer.cs)). Only the span bridge shipped. Disposition below |
| 3 | `TrimDirectiveLines` default flip | `false` | `true` | ✅ Shipped |
| 4 | `[Obsolete]` on `AllowCSharp` (window decision W1) | Un-attributed bridge over `ExpressionMode` | `[Obsolete]`, behavior unchanged | ❌ **Slipped** — no attribute in 2.0.0; the XML doc still says "Scheduled". Disposition below |
| 5 | Generator MSBuild defaults follow the engine | `HeddleOutputProfile=Text`, `HeddleTrimDirectiveLines=false` | `Html` / `true` | ✅ Shipped ([Heddle.Generator.props](../../src/Heddle.Generator/build/Heddle.Generator.props)) |

**Window decisions that stand as shipped:**

- **W2 — precompiled assemblies from 1.x fall back under a 2.0 engine.** No compatibility
  shim: the engine-version gate rejects 1.x manifests; outcome follows
  `PrecompiledMismatchPolicy` (`Fallback` recompiles with `HED7101`, `Strict` throws).
  Shipped and documented in the [CHANGELOG](../../CHANGELOG.md) and
  [precompilation.md](../precompilation.md). Rationale preserved: the gate exists so a
  byte-changing window can never be masked by stale generated code.

**Disposition of the slipped items** (full evidence in the
[ledger entry](#cross-spec-amendments-ledger) below):

- **Item 4 → 2.1, additively.** `[Obsolete]` is warning-only and needs no window;
  [improvement-spec B1](../improvement-spec.md#b1--obsolete-on-templateoptionsallowcsharp)
  owns it. W1's original rationale (fold into the single 2.0 migration touch) is moot —
  that touch has happened.
- **Item 2, pluggability half → 2.1, additively.** `TemplateOptions.Encoder` ships with
  the ratified `TextEncoder` seam but a **byte-identical default** (`null` = the 1.x/2.0
  `WebUtility` behavior) — [improvement-spec B2](../improvement-spec.md#b2--templateoptionsencoder)
  owns it.
- **Item 2, default-swap half → the 3.0 register.** Swapping the *default* encoder to
  `HtmlEncoder.Create(UnicodeRanges.All)` changes rendered bytes and its licensed window
  has closed; it is a candidate in the
  [next-window register](common/breaking-windows.md#next-window-candidate-register).

**Explicitly excluded from the 2.0 window** (decisions preserved from the original window
spec): `[NotEncode]` type deletion (needs its own ratified window — now a next-window
candidate); `TypeForwardedTo` hygiene for `Heddle.LanguageServices` (churn without
benefit); async render APIs (sync-sinks decision stands); any grammar change (everything
shipped additively at the grammar level).

**Verification gate (as executed):** the standard regression gate plus
`samples/html-safe-output` running under both profile defaults, and the precompiled ==
dynamic differential against the 2.0 defaults proving engine and generator flipped
together.
